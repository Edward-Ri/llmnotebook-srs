## Phase 3.2：Reference 面板布局重构

> 对应开发流程 Phase 3.2（Milestone A++ 体验优化）。本文件描述将 Reference 面板从"头部臃肿 + 内嵌二分"布局重构为"可收缩来源侧边栏 + 全宽阅读区 + 内嵌关键词 + 浮动操作条 + 右侧滑出校验面板"的三区布局完整技术方案。

---

### 1. 背景与动机

#### 1.1 当前布局问题

Phase 3 落地的 Reference 面板（`reference-panel.tsx`）存在以下体验问题：

- **头部信息冗余**："Reference 阅读区"标签 + "原文与关键词"标题 + 描述文案 + 导入按钮 + Reference 切换 Pills + 选中 Reference 信息卡，合计消耗约 180–200px 纵向空间。
- **左栏内部二分**：原文段落与目录/关键词各占约 60:40 水平空间，导致原文阅读区在屏幕上仅约总宽度的 35%（58% × 60%），可读性差。
- **关键词展示与阅读混合**：大量关键词 Badge 和 outline 卡片抢占了与原文并列的空间，但关键词选择是低频操作，不需要常驻可见。
- **校验遮挡阅读**：候选卡片校验使用底部 Sheet，打开后遮挡整个阅读区，无法对照原文审核。

#### 1.2 设计参考

参考 Google NotebookLM 的三栏布局：

- 左侧可收缩"来源"侧边栏（仅放来源列表 + 导入按钮）。
- 中部全宽内容阅读区。
- 右侧 Studio/Notebook 面板。

#### 1.3 目标效果

- 原文阅读区获得最大化展示宽度，提升可读性。
- Reference 切换和导入收进可收缩侧边栏，不占用阅读空间。
- 关键词与目录融入段落流，作为折叠式章节分隔出现，按需展开。
- 候选卡片校验改为右侧滑出面板，可与原文并排对照。

---

### 2. 目标布局

#### 2.1 桌面端三区布局

```
┌──────┬──────────────────────────────┬──────────────────┐
│ 来源  │  阅读区（原文 + 内嵌关键词）  │  Notebook         │
│侧边栏 │                              │  (右栏)           │
│(可收缩)│                              │                  │
└──────┴──────────────────────────────┴──────────────────┘
```

- **左侧边栏**：固定宽度 ~260px，可收缩至 ~48px。
- **阅读区**：占据剩余主体宽度，全宽展示原文。
- **右栏**：保留现有 Notebook 设计，通过 `ResizableHandle` 可调节与阅读区的比例。

#### 2.2 侧边栏收缩后

```
┌──┬──────────────────────────────────────┬──────────────────┐
│☰ │  阅读区（全宽）                        │  Notebook         │
└──┴──────────────────────────────────────┴──────────────────┘
```

#### 2.3 校验面板打开时

```
┌──────┬──────────────────────────────┬──────────────────┐
│ 来源  │  阅读区（可对照原文）          │ 候选卡片校验  [✕] │
│      │                              │  卡片 1 / 卡片 2  │
│      │                              │  [保留][编辑][丢弃]│
└──────┴──────────────────────────────┴──────────────────┘
```

校验面板临时替换 Notebook 面板，关闭后 Notebook 恢复。

### 3. 来源侧边栏

#### 3.1 组件

新建 `reference-source-sidebar.tsx`。

#### 3.2 Props

```typescript
interface ReferenceSourceSidebarProps {
  isOpen: boolean;
  onToggle: () => void;
  references: WorkspaceReference[];
  selectedReferenceId: string | null;
  onSelectReference: (referenceId: string) => void;
  onOpenImport: () => void;
  onDeleteReference: (reference: WorkspaceReference) => Promise<void>;
  isLoading: boolean;
}
```

#### 3.3 展开状态（~260px）

- 顶部："来源"标签 + 收起图标按钮（`PanelLeftClose`）。
- "+ 导入 Reference"按钮。
- Reference 列表：每项为 `button`，左侧图标 + 截断标题（`max-w-[180px] truncate`），选中项高亮（`bg-primary text-primary-foreground`）。hover 时显示删除图标。
- 加载态：`Skeleton` 占位。
- 空态："还没有导入任何 Reference"提示文案。

#### 3.4 收起状态（~48px）

- 仅显示展开图标按钮（`PanelLeftOpen`）。
- 内部内容通过 `overflow-hidden` 隐藏。

#### 3.5 过渡动画

- 使用 CSS `transition-[width] duration-200 ease-in-out` 实现平滑宽度过渡。
- 不使用 shadcn 全局 `Sidebar` 组件（它采用 `fixed` 定位 + `min-h-svh`，不适合嵌入式工作区布局），而是自建轻量 `<aside>` 元素。

### 4. 阅读区

#### 4.1 组件

重构现有 `reference-panel.tsx`，去除头部/切换器/信息卡/内嵌二分，改为全宽阅读区。

#### 4.2 Props 变更

移除（已迁移到侧边栏）：
- `references`、`onSelectReference`、`onOpenImport`、`onDeleteReference`、`isReferencesLoading`

新增：
- `selectedReference: WorkspaceReference | null` -- 当前选中的 Reference 完整对象
- `pendingCardCount: number` -- 待审核候选卡片数
- `onOpenValidation: () => void` -- 打开校验面板

#### 4.3 顶部标题栏

固定在阅读区顶部，不随内容滚动：

```
┌─────────────────────────────────────────────────────┐
│  Reference 标题 · 5段原文 · 16个关键词  [🗑]  📋12  │
└─────────────────────────────────────────────────────┘
```

- 左侧：当前 Reference 标题 + 统计信息（段数/关键词数）+ 删除按钮（`Trash2` 图标）。
- 右侧：待审核卡片 Badge，仅 `pendingCardCount > 0` 时显示，点击触发 `onOpenValidation`。
- 无选中 Reference 时：显示空态提示"请在侧边栏中选择一份 Reference"。

#### 4.4 段落流（ScrollArea 全宽）

保留现有段落渲染逻辑，每个段落 `<article>` 保留 `draggable` 属性和选区浮动工具栏。

**新增：章节分隔标题**

在每组段落**之后**插入章节分隔，使用 `Collapsible` 组件（已有 `collapsible.tsx`）：

```
第 1 段 ...
第 2 段 ...
第 3 段 ...
第 4 段 ...

▶ 段落 1-4 · 信息检索概论              ← 折叠（默认状态）

第 5 段 ...

▶ 段落 5 · 算法实现细节
```

展开后显示该节关键词 Badge：

```
▼ 段落 1-4 · 信息检索概论              ← 展开
┌────────────────────────────────┐
│ [✓艾宾浩斯遗忘曲线] [大语言模型]│
│ [记忆卡片] [✓间隔重复系统]      │
└────────────────────────────────┘
```

**渲染逻辑**：

遍历 `flatOutline`，为每个 outline node 在其 `endIndex` 段落之后插入 `Collapsible` 分隔组件。具体实现：

1. 构建一个 `Map<number, FlatOutlineNode[]>`，key 为 `endIndex`，value 为在该段落后需要插入的分隔列表。
2. 渲染段落时，在每个段落之后检查是否有对应 `endIndex` 的分隔需要插入。
3. 分隔组件使用 `Collapsible` + `CollapsibleTrigger`（章节标题行）+ `CollapsibleContent`（关键词 Badge 列表）。

#### 4.5 底部浮动操作条

条件渲染，仅 `selectedKeywordIds.length > 0` 时显示：

```
┌──────────────────────────────────────┐
│  已选 3 个关键词    [生成候选卡片]    │
└──────────────────────────────────────┘
```

- 定位：`sticky bottom-0`，贴在 ScrollArea 底部。
- 样式：半透明背景 + `backdrop-blur-md` + 圆角 + 边框。
- "生成候选卡片"按钮：点击后变为 spinner + "生成中..."，生成完成后 toast 通知。
- 未选中关键词时完全隐藏，阅读区干净纯粹。

### 5. 校验面板

#### 5.1 交互流程

1. 用户点击阅读区顶部标题栏的待审核 Badge（如 `📋12`）。
2. 右栏 Notebook 面板平滑向右滑出，校验面板从右侧滑入，占据相同位置和宽度。
3. 用户在校验面板中审核卡片，同时可查看左侧阅读区原文对照。
4. 点击校验面板顶部 [✕] 关闭按钮。
5. 校验面板向右平滑滑出，Notebook 面板从右侧平滑滑回。

#### 5.2 实现方式

在 `material-detail.tsx` 中，右栏 `ResizablePanel` 内根据 `isValidationOpen` 状态条件渲染：

```tsx
<ResizablePanel>
  {isValidationOpen
    ? <CardValidationPanel onClose={() => setIsValidationOpen(false)} />
    : <NotebookPanel ... />}
</ResizablePanel>
```

过渡动画使用 CSS `transition` + `transform: translateX`，或简单的 opacity 淡入淡出。

#### 5.3 校验面板内容

- 顶部：标题"候选卡片校验" + 关闭按钮。
- 主体：可滚动的卡片列表，每张卡片显示 Q/A 内容，提供「保留」「编辑」「丢弃」操作。
- 复用现有 `DocumentCardValidation` 组件逻辑。

#### 5.4 移动端

移动端屏幕宽度不足以并排展示，回退为底部 Sheet（保留现有 `Sheet` 实现）。

### 6. 页面布局重构

#### 6.1 桌面端 material-detail.tsx 布局

```tsx
<div className="flex h-full">
  <ReferenceSidebar
    isOpen={isSidebarOpen}
    onToggle={() => setIsSidebarOpen(v => !v)}
    references={references}
    selectedReferenceId={selectedReferenceId}
    onSelectReference={setSelectedReferenceId}
    onOpenImport={() => setIsImportOpen(true)}
    onDeleteReference={handleDeleteReference}
    isLoading={referencesQuery.isLoading}
  />

  <ResizablePanelGroup direction="horizontal" className="flex-1">
    <ResizablePanel defaultSize={58} minSize={45}>
      <ReferencePanel
        selectedReference={selectedReference}
        blocks={blocks}
        outline={outline}
        selectedKeywordIds={visibleSelectedKeywordIds}
        pendingCardCount={pendingCardCount}
        isBlocksLoading={referenceBlocksQuery.isLoading}
        isOutlineLoading={referenceOutlineQuery.isLoading}
        isGenerating={isGenerating}
        canGenerate={visibleSelectedKeywordIds.length > 0}
        onToggleKeyword={handleToggleKeyword}
        onGenerateCards={handleGenerateCards}
        onOpenValidation={() => setIsValidationOpen(true)}
        onSendSelectionToNotebook={handleSendSelectionToNotebook}
      />
    </ResizablePanel>
    <ResizableHandle withHandle />
    <ResizablePanel defaultSize={42} minSize={32}>
      {isValidationOpen
        ? <CardValidationPanel onClose={() => setIsValidationOpen(false)} documentId={id} />
        : <NotebookPanel ... />}
    </ResizablePanel>
  </ResizablePanelGroup>
</div>
```

#### 6.2 新增状态

```typescript
const [isSidebarOpen, setIsSidebarOpen] = useState(true);
```

`isValidationOpen` 现有状态保留，但其触发来源从底部 Sheet 按钮改为阅读区顶部 Badge。

#### 6.3 移动端

- 侧边栏隐藏，Reference 选择改为阅读区顶部的 dropdown/select 组件。
- 校验面板使用底部 Sheet（保留现有实现）。
- 阅读区和 Notebook 垂直堆叠。

### 9. 与现有功能的兼容

- **拖拽到 Notebook**：段落 `draggable` 属性和 `dragstart` 数据格式不变，Notebook 编辑器的 drop 接收逻辑不变。
- **选区浮动工具栏**：`selection-toolbar.tsx` 保留，在全宽阅读区中继续工作。
- **关键词选择**：原有 `onToggleKeyword` 逻辑不变，只是关键词 Badge 从独立右侧区域迁移到段落内嵌 Collapsible 中。
- **卡片生成**：`handleGenerateCards` 逻辑不变，触发入口从固定工具栏改为浮动操作条。
- **候选卡片校验**：`DocumentCardValidation` 组件逻辑不变，容器从底部 Sheet 改为右栏面板（桌面端）。
